# Swan's Office --- Creative Design Bible (Version 1)

> **One sentence pitch**
>
> *A portfolio you don't scroll --- you visit.*

------------------------------------------------------------------------

# 1. Purpose

This document is the complete creative blueprint for the portfolio.

It should be understandable by: - Designers - Developers - Artists -
Recruiters - Anyone with zero technical knowledge

Reading this document should allow someone to understand the entire
vision without seeing a prototype.

------------------------------------------------------------------------

# 2. Core Idea

The portfolio is **one living office**.

The camera never moves.

The visitor feels like they are standing in front of Sowan's desk.

A duck (Swan) is working there.

He notices visitors.

He talks.

He reacts.

He becomes the guide to the portfolio.

The office itself is alive.

This is not a game.

This is not a website.

It is an interactive animated experience.

------------------------------------------------------------------------

# 3. Origin Story

People often misread **Sowan** as **Swan**.

Instead of correcting them...

The joke becomes the brand.

The duck introduces himself:

> "Hi! I'm Swan."

Small pause.

> "...Well... actually I'm Sowan."

> "People keep calling me Swan."

> "So I decided to embrace it."

This immediately makes the visitor smile.

# 5. Camera

One fixed camera.

Never rotates.

Never pans.

Never zooms.

This makes:

-   mobile easier
-   desktop easier
-   development easier
-   storytelling stronger

The duck is always visible.

------------------------------------------------------------------------

# 6. Duck Personality

The duck is:

-   confident
-   calm
-   funny
-   slightly sarcastic
-   welcoming
-   intelligent

Never arrogant.

Never acts like an AI chatbot.

Feels like meeting an engineer in their office.

-   sleeping
-   blinking
-   checking the clock
-   stretching
-   looking outside
-   adjusting pipe
-   quietly humming

The office continues breathing.

------------------------------------------------------------------------

# 8. Interaction Philosophy

Every interaction follows the same pattern.

User clicks.

↓

Duck notices.

↓

Duck looks directly at the visitor.

↓

Dialogue appears.

↓

Relevant object reacts.

↓

Portfolio content opens.

↓

User closes content.

↓

Object returns.

↓

Duck resumes previous activity.

Continuity is important.

Nothing opens like a normal website.

------------------------------------------------------------------------

# 12. Visual Production Plan

We will not hand-draw hundreds of assets.

Pipeline:

1.  Define ONE art style.
2.  Generate concept art.
3.  Generate individual props.
4.  Clean and unify them.
5.  Animate.

Suggested tools:

-   ChatGPT image generation for concepts and consistent prompts.
-   Claude + Codex for implementation.
-   Pixel editor (Aseprite or similar) if manual touch-ups are needed.
-   Sound libraries for ambience.

Every generated asset must match the same style guide.

------------------------------------------------------------------------

# 15. Mobile First

The portfolio must work beautifully on phones.

Rules:

-   Fixed scene scales proportionally.
-   Tap replaces hover.
-   Clickable objects remain large enough.
-   Dialogue never covers the duck.
-   Book/folder UI becomes full-screen on mobile.
-   Animations stay lightweight.
-   Fast loading is mandatory.

Desktop gains extra detail but not extra functionality.
